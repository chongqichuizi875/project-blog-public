# 第一部分 · 方法与效果

## 1. 背景：统一的 token-level trust region 与自回归结构不匹配

RLVR（带可验证奖励的强化学习）已成为提升 LLM 推理能力的标准做法：策略生成回复、verifier 给出标量奖励、再用 PPO/GRPO 式的 token 级目标更新模型。实际训练是**离策略（off-policy）**的——每一批回复由固定的 rollout 策略 $\mu$ 采样、复用若干个梯度步，被优化的策略 $\pi$ 因此持续偏离采样它的策略。不加约束的更新会破坏训练稳定性、损害推理性能。

为抑制这种漂移，RLVR 沿用了经典策略优化中的 **trust region** 思想：

- **PPO / GRPO** 对采样 token 的似然比 $\rho_t = \pi(y_t\mid s_t)/\mu(y_t\mid s_t)$ 做裁剪。但单 token 的比率是真实散度的**单样本蒙特卡洛估计**，方差高、且在长尾词表上标定不良。
- **DPPO** 把采样比率换成对 token 级散度 $D_t = D_{\mathrm{TV}}(\mu(\cdot\mid s_t),\,\pi(\cdot\mid s_t))$ 的直接度量，反映了下一 token 分布的真实变化，而非单次采样结果。

这些方法改进了**如何度量** token 级散度，却都对所有位置施加一个**统一、位置无关的阈值** $\delta$。这与 LLM 的自回归分解存在两处结构性冲突：

1. **统一阈值忽略了误差传播的自回归不对称性。** 早期 token 决定了后续全部生成的条件，因此相同的 token 级散度发生在越早的位置，引起的序列级分布偏移越大。统一阈值因而**低估早期偏差、又过度限制晚期探索**。
2. **孤立地评估 token 级散度，忽略了累积前缀漂移。** 第 $t$ 步的状态就是已生成前缀 $s_t = (x, y_{<t})$。若逐 token 偏差沿前缀累积，模型实际是在一个高度离策略的状态上生成。当条件前缀已经显著偏离 rollout 策略时，当前 token 的任何进一步偏离都带有复合风险——而统一阈值无论前缀已漂移多远，都给出**完全相同**的散度额度。

## 2. 方法：位置加权阈值 + 累积前缀预算

CPPO 不改变散度的度量方式，只**重新分配**沿回复允许的散度额度，对应上面两处缺陷给出两个耦合的机制。

### 2.1 位置加权的 token 级阈值

引入一个随位置**单调递减**的权重 $w_t$，并把约束施加在加权散度 $w_t D_t$ 上。实现采用线性 schedule：

$$
w_t = 1 - \frac{1 - w_{\min}}{T-1}(t-1), \qquad w_t \in [w_{\min}, 1].
$$

由于约束作用在 $w_t D_t \le \delta$ 上，它等价于一个**位置相关**的 token 级阈值 $D_t \le \delta / w_t$：在回复开头为 $\delta$，到结尾放松至 $\delta / w_{\min}$。早期位置因而只允许更小的 $D_t$，晚期位置——remaining horizon 更短、下游影响更小——被适当放松。

### 2.2 累积前缀预算

记加权前缀散度与累积权重

$$
S_t = \sum_{j=1}^{t} w_j D_j, \qquad W_t = \sum_{j=1}^{t} w_j, \qquad S_0 = W_0 = 0.
$$

给定 token 级阈值尺度 $\delta$ 与前缀均值阈值 $\delta_b$，CPPO 只在如下条件成立时，保留一个把采样比率推离 1 的更新项：

$$
w_t D_t \;\le\; c_t^{\mathrm{CPPO}} := \min\!\Big\{\, \delta,\;\; \delta + \delta_b W_{t-1} - S_{t-1} \,\Big\}.
$$

这一**有效阈值**同时强制 token 级条件 $w_t D_t \le \delta$ 与累积前缀条件 $S_t \le \delta + \delta_b W_{t-1}$（额外的 $\delta$ 项给首个 token 完整额度）。其行为是**动态的**：当前缀加权平均 $S_{t-1}/W_{t-1}$ 仍低于 $\delta_b$ 时，token 级阈值生效；一旦先前偏差把前缀均值推过 $\delta_b$，前缀项 $\delta + \delta_b W_{t-1} - S_{t-1}$ 降到 $\delta$ 以下、收紧当前 token 的额度。**trust region 随前缀漂移累积而动态收紧**，而非给每个 token 同样的额度。

### 2.3 合成为单个 token 级掩码

两个条件合并为单个逐 token 判据 $I_t$，套进标准的 token 级 ratio-advantage 目标（advantage 仍用 GRPO 的 group-relative advantage）：

$$
M_t^{\mathrm{CPPO}} = \mathbf{1}\!\big[\, \hat A_t(\rho_t - 1) \le 0 \;\;\vee\;\; I_t \,\big], \qquad I_t:\ w_t D_t \le \delta \ \wedge\ S_t \le \delta + \delta_b W_{t-1}.
$$

第一项保留所有把 $\pi$ 拉近 $\mu$ 的更新；$I_t$ 只约束把 $\pi$ 推离 $\mu$ 的项。$S_t, W_t$ 都是普通前缀和，批量实现用一次沿回复维的 cumsum + 一次右移一位即可，**无额外前向、无新 loss 项**——这使 CPPO 与 DPPO 在工程上几乎零差异，从而保证后文的对比严格隔离出「散度如何分配」这一变量。

# 第二部分 · 理论与消融分析

> 本部分回答两个问题：**① 为什么位置加权 + 前缀预算能给出比统一阈值更紧的 improvement bound（理论）；② 两个机制各自贡献多少、对散度度量/近似是否鲁棒（消融）。**

## 4. 理论：比同期工作更紧的 monotonic improvement bound

### 4.1 残差从何而来

固定 rollout 策略 $\mu$、在公共支撑上优化 $\pi$。记 $\rho_{a:b} = \prod_{j=a}^{b}\rho_j$。自回归分解给出**精确的 finite-horizon 性能差分恒等式**：

$$
J(\pi) - J(\mu) = L'_\mu(\pi) - \Delta(\mu, \pi),
$$

其中 token 级 surrogate 目标 $L'_\mu(\pi) = \mathbb{E}_\mu\big[R\sum_t(\rho_t - 1)\big]$，而残差

$$
\Delta(\mu, \pi) = \mathbb{E}_\mu\Big[R\sum_{t=1}^{T}(\rho_t - 1)\,(1 - \rho_{t+1:T})\Big].
$$

trust region 真正要控制的是这个残差 $|\Delta|$——它来自被 surrogate 目标丢弃的**suffix correction** $1 - \rho_{t+1:T}$：第 $t$ 个 token 的偏差，会进入它之后**每一个** token 的条件。

### 4.2 remaining-horizon 放大

设逐 token 散度满足阈值轮廓 $D_t \le \ell_t$，$u_t := \mathbb{E}[D_t]$ 为位置 $t$ 的期望散度。对后缀似然比做最大耦合可得残差界：

$$
|\Delta(\mu, \pi)| \;\le\; 4\xi \sum_{t=1}^{T-1} u_t \sum_{j=t+1}^{T}\ell_j \;\le\; \sum_{t=1}^{T-1}\lambda_t u_t, \qquad \lambda_t := 4\xi\,\bar\ell\,(T - t).
$$

系数 $\lambda_t \propto (T - t)$ 随 **remaining horizon 线性增长**——这把第一处缺陷量化了：一次早期偏差把生成前缀的分布整体偏移，扭曲后续所有 token 的条件，其残差误差在整个后缀上累积。统一阈值 $D_t \le \delta$ 完全忽略这一轮廓，于是**低估携带最大 remaining-horizon 乘子的早期偏差、又过度惩罚几乎无下游影响的晚期偏差**。

### 4.3 前缀约束把 bound 收紧

CPPO 通过加权前缀均值施加约束 $w_t D_t \le c_t$、$P_m \le \delta_b W_m$。其 policy-improvement bound 为：

$$
J(\pi) - J(\mu) \;\ge\; L'_\mu(\pi) - 2\xi T(T-1)\,\bar\ell\,\delta_b.
$$

**为什么 Abel summation 能把 bound 收紧。** 残差被界为 $\sum_{t<T}\lambda_t u_t$，其中 $\lambda_t\propto(T-t)$ 递减、$u_t=\mathbb{E}[D_t]$。逐点约束只能给出 $u_t\le\delta$，于是只能得到 $\sum_t\lambda_t u_t\le\delta\sum_t\lambda_t$——这等于放任策略在**每一个连续步**都顶满散度额度，而这种最坏累积恰恰把 $\pi$ 的状态访问推离 $\mu$、直接抬高 $|\Delta|$。CPPO 改为约束加权前缀**均值** $S_m=\sum_{j\le m}w_j(u_j-\delta_b)\le 0$。令 $r_t:=\lambda_t/w_t$，Abel summation（离散分部求和）把同一个和改写为

$$
\sum_{t<T}\lambda_t u_t - \delta_b\sum_{t<T}\lambda_t \;=\; r_{T-1}S_{T-1} + \sum_{t=1}^{T-2}(r_t - r_{t+1})\,S_t.
$$

只要 position-weight schedule 使 $r_t$ 单调不增（线性 $w_t$ 满足，见原文 Prop.），右侧每一项都是「非负系数 $(r_t-r_{t+1})$ × 非正 prefix slack $S_t$」，因此整体 $\le 0$，得到 $\sum_t\lambda_t u_t\le\delta_b\sum_t\lambda_t$。直觉上，Abel summation 把「逐点顶满」的最坏路径换成了「prefix 均值受限」的约束族——prefix 越早、$\lambda_t$ 越大的位置被压得越紧，于是 $\delta$ 被替换为更小的 prefix 均值 $\delta_b$，这正是 bound 收紧的来源（取 $u_t\equiv\delta_b$ 时取等，故对这族约束是紧的）。

**与同期工作的 bound 对照。** 把 $\sum_{t<T}\lambda_t=2\xi T(T-1)\bar\ell$ 代入，各方法的残差常数为：

- **vs DPPO（uniform token-level trust region）**：DPPO 把 $\delta$ 同时用作 $\bar\ell$ 与 $u_t$ 的上界，常数为 $\delta^2$；CPPO 保留同一个 $\bar\ell=\delta$，但前缀约束把 $u_t$ 项的总和压到 $\delta_b$，得到 $\delta\cdot\delta_b$。两者之比 $C_{\mathrm{CPPO}}/C_{\mathrm{DPPO}}=\delta_b/\delta$，在 $\delta_b<\delta$ 时严格更紧。
- **vs TRM-Max**：$\max_t D_t\le\delta$ 蕴含每个 $u_t\le\delta$，与 DPPO 落到完全相同的二次常数 $\delta^2$——序列级取 max 并不能改善逐点最坏累积。
- **vs TRM-Avg**：只在**整条回复的终端**约束一次均值，等价于只保留最后一个前缀约束。直接计算给 $C_{\mathrm{terminal}}\le 4\xi\bar\ell(T-1)^2\delta_b$，而 CPPO（$w\equiv1$ 时）用上**每一个**前缀得到 $2\xi T(T-1)\bar\ell\delta_b$，两者之比为 $2-\tfrac{2}{T}$——即终端约束最坏会松约 2 倍，因为它允许早期前缀大幅超标、再靠后段拉回平均。CPPO 在每个 $m$ 都强制 $S_m\le 0$，杜绝了这种「早超标、晚补偿」。

> 关键判断：这三条对照说明，CPPO 的更紧并非来自更小的阈值或更强的散度度量，而是来自**把约束施加在每个前缀的加权均值上**——Abel summation 正是把这一约束结构转化为更小残差常数的工具。位置加权部分进一步给出 $C_{\mathrm{CPPO}}/C_{\mathrm{uniform}}\le \delta_b/(\delta w_{\min})$，其中 $1/w_{\min}$ 是放松晚期阈值的代价。理论收紧的方向（约束累积漂移、对早期收紧）与 §3 实测增益在大模型 / 长 horizon 上最显著、与 DPPO 差距随训练扩大的现象互证。

三条结论收敛到同一点：CPPO 的增益**来自如何沿回复分配散度预算（位置顺序 + 前缀累积），而非散度的度量方式或近似粒度**。这与 §4 的理论一致——收紧的是残差常数中「逐点 $\delta$ → 前缀均值 $\delta_b$」这一项，与具体用 TV 还是 KL、Top-K 还是 Binary 无关。
